---
title: "User creation"
api: "POST https://api.keywordsai.co/api/users/"
---

Creates a new customer (or upserts an existing customer).

## Authentication

<Info>
Use either:

- `Authorization: Bearer YOUR_API_KEY` (public API)
- `Authorization: Bearer YOUR_JWT_TOKEN` (platform/internal)
- `X-Auth-Api-Key: YOUR_API_KEY` (alternative header)
</Info>


<ParamField path="customer_identifier" type="string" required={true}>
    Your unique identifier for this customer.
</ParamField>

<ParamField path="name" type="string">
    Customer's display name.
</ParamField>

<ParamField path="email" type="string">
    Customer's email address.
</ParamField>

<ParamField path="period_budget" type="number">
    Spending limit for the current budget period in USD. Use `null` to remove the budget limit.
</ParamField>

<ParamField path="budget_duration" type="string" default="monthly">
    Budget renewal period. Options: `daily`, `weekly`, `monthly`.
</ParamField>

<ParamField path="environment" type="string" default="prod">
    Environment. Options: `prod`, `test`.
</ParamField>

<RequestExample>
```Python Python
import requests

url = "https://api.keywordsai.co/api/users/"
api_key = "YOUR_API_KEY"
data = {
    "customer_identifier": "user_123",
    "email": "john@example.com",
    "name": "John Doe",
    "period_budget": 100.0,
    "budget_duration": "monthly"
}
headers = {
    "Authorization": f"Bearer {api_key}",
    "Content-Type": "application/json"
}

response = requests.post(url, headers=headers, json=data)
print(response.json())

```

```TypeScript TypeScript
fetch("https://api.keywordsai.co/api/users/", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_API_KEY"
  },
    body: JSON.stringify({
        customer_identifier: "user_123",
        email: "john@example.com",
        name: "John Doe",
        period_budget: 100.0,
        budget_duration: "monthly"
    })
})
.then(response => response.json())
.then(data => console.log(data));
```
</RequestExample>

<Note>
If a customer exceeds their budget, requests may be rejected with a `402` error.
</Note>
